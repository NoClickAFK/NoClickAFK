# «Богатка» 4.0.0 — матрица реализации

Дата сборки: 2026-06-26

## Принцип хранения

Все новые данные сохраняются внутри `locations.form_data` и локальной IndexedDB. Отдельная миграция структуры Supabase не требуется. Облачная синхронизация, резервная копия, локальный HTML/PDF и публичный отчёт используют одну структуру данных.

## Интеграции

- **Облако:** новые структуры передаются внутри `form_data` существующей таблицы `locations`.
- **Конкурентное редактирование:** задачи, комментарии, история и этапы запуска объединяются по UUID; удаления хранят tombstone-идентификаторы.
- **Роли:** `viewer` получает режим только для чтения; `editor` и `owner` могут изменять данные.
- **Резервная копия:** формат версии 4 включает активные и архивные локации, историю, задачи, комментарии, экономику, запуск и оригиналы фотографий.
- **Очистка:** общая очистка затрагивает только активные локации; архив сохраняется.
- **Отчёты:** HTML, PDF и публичная ссылка включают экономику, стоп-факторы, фотоплан, задачи, комментарии, историю и проект запуска.
- **Офлайн:** новые JS/CSS загружаются через существующий network-first Service Worker и после первого онлайн-запуска доступны из кэша.

## Встроенные проверки

При запуске приложение проверяет наличие основных модулей, сумму весов (100), расчёт аренды за м² и арендной нагрузки. Результат выводится в шапке как `Самопроверка: OK` или количество найденных ошибок и сохраняется в `localStorage`.

## Ограничение приёмки

Статический и встроенный программный контроль не заменяет финальный пользовательский проход на реальных iPhone/Android/desktop-браузерах. Версия 4.0.0 считается готовой к комплексному приёмочному тесту после публикации GitHub Pages и появления `Самопроверка: OK`.
